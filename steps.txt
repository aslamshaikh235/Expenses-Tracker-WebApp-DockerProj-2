Step 2 - CREATE AWS INFRASTRUCTURE (Terraform)

Create a folder Infra/terraform/

Infra/terraform/
│
├── provider.tf
├── vpc.tf
├── subnets.tf
├── igw.tf
├── route-table.tf
├── variables.tf
├── outputs.tf

👉 Terraform automatically reads all .tf files together
👉 So you don’t need to “connect” them manually

🧠 WHY THIS STRUCTURE IS BEST
File	Purpose
provider.tf	    AWS config
variables.tf	reusable values
vpc.tf	        core network
subnets.tf	    network layers
igw.tf	        internet
route-table.tf	traffic routing
outputs.tf	    debugging

🧠 HOW TO IDENTIFY WHAT IS NEEDED?

This is the most important question you asked 👇

🎯 Think from FINAL GOAL

You want:

User → ALB → Kubernetes (EKS) → App
👉 Then break it down:
1. Need network?

✔ Yes → VPC

2. Need internet access?

✔ Yes → IGW + Public Subnets

3. Need secure backend?

✔ Yes → Private Subnets

4. Need load balancer?

✔ Yes → Public Subnets (2 AZ required)

5. Need Kubernetes?

✔ Yes → Private Subnets (best practice)

👉 That’s how DevOps engineers think
